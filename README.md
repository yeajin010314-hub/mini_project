프로젝트 개요 (Overview)

프로젝트 명
여행지 / 맛집 추천을 위한 기사 등록 시스템

한 줄 소개
여행지 및 맛집 정보를 기사 형태로 등록하고 카테고리 기반으로 조회할 수 있는 웹 콘텐츠 관리 시스템입니다.

개발 기간
2026.01.21 ~ 2026.02.20 (약 1개월)

3명 (팀 프로젝트)

● 주요 기능 (Features)
1. 회원 관리 기능
- 회원가입 기능 구현 및 입력 데이터 검증 처리
- 로그인 기능 구현 (세션 기반 인증 처리)
- 로그인 상태 유지 및 세션 기반 사용자 정보 관리
- 작성자 본인 여부 검증을 통한 권한 제어 로직 구현

2. 인증 및 접근 제어
- HttpSession을 활용한 로그인 상태 관리
- 비로그인 사용자 접근 시 로그인 페이지로 리다이렉트 처리
- 게시글 수정/삭제 요청 시 세션 사용자와 작성자 비교 검증
- URL 직접 접근 시 서버단에서 접근 차단 처리

3. 서버 요청 처리 로직 구현
- Servlet을 활용한 클라이언트 요청 처리
- 요청 파라미터 수집 및 DAO 호출 구조 구현
- 처리 결과를 JSP(View)로 전달하는 흐름 구성
- 예외 상황에 따른 분기 처리 및 오류 대응 로직 구현

4. 데이터 처리 및 DB 연동
- DAO 패턴을 활용한 데이터 접근 로직 구현
- Oracle DB와 연동하여 회원 정보 조회 및 저장 처리
- 회원 상태 코드 기반 관리 구조 적용 

※시스템 구조 (Architecture)

본 프로젝트는 MVC(Model-View-Controller) 패턴 기반 구조로 설계되었습니다.
Controller, DAO, View의 역할을 분리하여 요청 처리 흐름을 명확히 하고 유지보수성을 고려하여 구현했습니다.

●Controller (Servlet)
- 클라이언트 요청 URL 매핑 및 요청 처리
- 요청 파라미터 수집
- DAO 호출 및 비즈니스 로직 처리
- 세션 기반 로그인 상태 및 권한 검증
- 처리 결과를 JSP(View)로 전달

●DAO (Data Access Object)
- Oracle DB 접근 및 SQL 실행
- 회원 데이터 조회 및 저장 처리
- 데이터 접근 로직 분리

●View (JSP)
- EL 및 JSTL을 활용한 데이터 출력
- 로그인 상태에 따른 UI 분기 처리
- 사용자 권한에 따른 기능 표시 제어


※구현 포인트
- Servlet 중심 서버 로직 구현 경험
- 세션 기반 인증 및 권한 제어 로직 구현
- 서버단 접근 제어를 통한 보안 강화
- MVC 구조에서 Controller → DAO → View 흐름 이해 및 구현

※트러블슈팅 (Troubleshooting)
URL 직접 접근 문제
☞ 문제
비로그인 사용자가 URL을 직접 입력하여 글쓰기 또는 수정 페이지에 접근할 수 있는 문제가 발생했습니다.

☞ 원인
초기 구현에서는 프론트엔드에서 버튼을 숨기는 방식으로만 접근을 제어하고 있었으며,
서버단에서 로그인 상태 검증이 충분히 이루어지지 않았습니다.

☞ 해결
Servlet에서 세션(Session) 존재 여부 확인
세션에 저장된 memberId 검증
로그인하지 않은 경우 로그인 페이지로 리다이렉트 처리

☞ 결과
비인가 사용자 접근 차단
서버단 권한 검증 로직 강화

☞배운 점

접근 제어는 UI가 아니라 서버단에서 반드시 검증되어야 한다는 점을 경험했습니다.
